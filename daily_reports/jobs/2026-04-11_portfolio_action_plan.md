## PHASE 2: 콘텐츠 보강 (4/13~4/16)

### 프로젝트별 점검 & 보강 항목

모든 기업에서 **designSystem이 핵심**이므로 이 프로젝트를 가장 먼저, 가장 탄탄하게 다듬습니다.

#### designSystem (최우선)

- [ ] **Problem**: 레거시 코드 문제의 정량 데이터 보강 (빌드 시간 22초 외 추가 지표 확보)
- [ ] **Strategy**: "왜 디자인 시스템이 해결책인가"의 논리 흐름 점검
- [ ] **SolutionA**: Cursor AI 활용한 중복 패턴 탐지 → 구체적 Before/After 코드량 시각화
- [ ] **SolutionB**: 가변형 그리드 시스템(12/8/4) 설명의 직관성 개선
- [ ] **Impact**: "서버 구동 22초→0.2초, 코드 40% 감축" 외 정성적 성과 추가 (개발팀 피드백 등)
- [ ] key.json 관련 키 텍스트 최종 검수

#### aiAgent (채널톡·클라썸 핵심)

### 기업별 경력 기술 튜닝

현재 RESUME_EXP_1_SUMMARY:
> "SaaS 기능 개선으로 검색량 26% 증대 및 디자인 시스템 구축으로 서버 가동 시간 22초 단축"

#### 채널톡 버전
> B2B SaaS 프로덕트 디자인 리드 — AI 에이전트 기획·프롬프트 설계, 디자인 시스템 구축으로 빌드 타임 98% 단축, 크로스펑셔널 팀과 빠른 이터레이션

**강조 순서**: AI 에이전트 → 디자인 시스템 → 개발 협업 → PO 겸직

#### 삼쩜삼 버전
> B2B SaaS 프로덕트 디자이너 — 디자인 시스템 구축(코드 40% 감축, 5.5만줄 레거시 제거), 복잡한 데이터 도메인의 서비스 플로우 전체 설계

**강조 순서**: 디자인 시스템 → 데이터 UX → 서비스 플로우 오너십

#### 당근마켓 버전
> B2B SaaS 프로덕트 디자이너 — 데이터 기반 UX 개선으로 검색량 26.4% 증대, 듀얼뷰 UI 특허 등록, 유저 리서치 주도

**강조 순서**: 데이터 기반 → 특허 → 유저 리서치 → 글로벌(일본어)

#### 클라썸 버전
> B2B SaaS 프로덕트 디자이너 — 리포트 작성 시간 99% 단축, 디자인 시스템 구축으로 개발 생산성 91배 향상, PM·개발팀 직접 협업

**강조 순서**: 임팩트 수치 → 개발 협업 → B2B SaaS 경력

## 핵심 원칙 리마인더

1. **designSystem 프로젝트가 전체 경쟁력의 핵심** — 가장 먼저, 가장 탄탄하게
2. **평가자는 훑어본다** — 핵심 수치를 크게, 과정 나열은 과감히 생략
3. **문제→전략→솔루션→임팩트** 논리가 끊기지 않도록
4. **JD 미러링** — 기업이 쓰는 용어를 내 포트폴리오에 그대로 반영
5. **비주얼보다 논리** — 채널톡·삼쩜삼 모두 "문제해결 사고방식"을 가장 중시
